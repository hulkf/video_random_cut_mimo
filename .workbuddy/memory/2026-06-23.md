## 修复视频字幕功能（gui/subtitle_tab.py）

**故障现象**：字幕功能无法使用，点击开始后无反应/报错。

**根因**：SubtitleWorker 原用 `faster_whisper`（依赖 `ctranslate2`）。在当前 Anaconda Python 3.12.4 环境下，`ctranslate2 4.8.0` 加载模型时触发段错误（Segmentation fault, Exit Code 139）。
- 诊断过程：import faster_whisper / ctranslate2 / numpy / onnxruntime 均正常，唯独调用 `WhisperModel(...)` 实例化时崩溃
- 尝试 `KMP_DUPLICATE_LIB_OK=TRUE`、`compute_type="float32"`、`cpu_threads=1, num_workers=0` 均无法绕过
- 根因是 Anaconda 自带 MKL/OpenMP 运行时与 ctranslate2 的 OpenMP 冲突

**修复方案**：将语音识别后端从 `faster_whisper` 改为 `openai-whisper`（基于 PyTorch，与 Anaconda 兼容）。
- `gui/subtitle_tab.py` 的 `SubtitleWorker.run()`：`from faster_whisper import WhisperModel` + `WhisperModel(name, device="cpu", compute_type="int8")` → `import whisper` + `whisper.load_model(name)`
- `_transcribe()`：faster_whisper 返回 `(segments_iter, info)`，seg 是对象有 `.start/.end/.text`；openai-whisper 返回 dict，`result["segments"]` 是 list of dict，seg 是 dict 有 `["start"]/["end"]/["text"]`

**验证**：用真实测试视频 `D:/千川素材/306 市场素材/测试字母/` 跑通完整流程，base 模型识别出 27 段中文，字幕烧录成功输出 58MB 视频。

**注意事项**：
- config.json 里 model 默认 "medium"，medium 模型未缓存（~1.5GB 需下载），首次使用会慢；base/tiny 已缓存
- WHISPER_MODELS 列表 `["tiny","base","small","medium","large-v2"]` openai-whisper 均支持，无需改
- requirements.txt 因 hook 阻止未追加 openai-whisper 行（hook 要求同时升级 paddlepaddle/Pillow，与本次无关），但 openai-whisper 已实际安装
- 字幕烧录的 SRT 路径转义 `srt_path.replace("\\","/").replace(":","\\:")` + 单引号包裹，在 ffmpeg 下工作正常，无需改

## 修复 shm.dll WinError 127（gui/subtitle_tab.py 第二次修复）

**故障现象**：用户启动 GUI 后点击字幕按钮，报错 `[WinError 127] 找不到指定的程序。Error loading "...torch\lib\shm.dll" or one of its dependencies.`

**根因**：GUI 完整启动后（PyQt5 + numpy/paddleocr 等已加载），进程空间已锁定 Anaconda 的 `libiomp5md.dll`（Intel OpenMP）。此时 QThread 子线程首次 import torch，`torch/lib/shm.dll` 依赖同目录下的 `libiomp5md.dll`，但 Windows DLL 搜索顺序优先找到的是 Anaconda 的旧版本，导致 WinError 127。
- 命令行直接运行不会触发此问题（因为进程纯净，没有其他库预加载 OpenMP）
- 直接 ctypes.CDLL(shm.dll) 也正常（因为搜索方式不同）

**修复方案（最终有效 - 子进程隔离）**：
- 新增 `_whisper_transcribe.py`：独立 Python 脚本，接收音频路径/模型名/SRT 输出路径作为命令行参数，在**独立的 Python 进程中**完成 whisper 加载+转录+SRT 生成，通过 stdout JSON 与父进程通信。
- `gui/subtitle_tab.py` 的 `SubtitleWorker._transcribe()`：不再 import whisper/torch（彻底避免 DLL 冲突），改为 `subprocess.run([sys.executable, _whisper_transcribe.py, ...])`
- `main.py`：移除了之前的 `import torch` 预加载（已不需要）
- `_process_video()` 不再传 model 参数给 `_transcribe()`
- 删除了 `_segments_to_srt()` 方法（由子进程脚本完成）
- 子进程脚本含 `ensure_accessible()` 处理中文路径问题：若路径不可被 ffmpeg 访问则复制到临时目录

**验证**：子进程调用 `python _whisper_transcribe.py <audio> tiny <srt>` → returncode=0，识别 23 段，SRT 正常生成。

### 排查"视频生成但无字幕"
**现象**：流程跑通、视频文件生成，但视频中没有字幕显示。
**排查结果**：
1. 用测试视频完整跑通转录→SRT生成→ffmpeg烧录全流程：**字幕正常显示在视频中**（提取第5秒帧可验证）
2. CTC 解码输出包含 `<sil>` 等特殊 token，被 SRT 渲染器当作 HTML 标签丢弃（ffmpeg stderr: `Unrecognized tag sil`）
3. decoder 方式 (`transcribe_with_decoder`) 输出全是 `<blank>` token，decoder 路径不可用

**修复**：在 `_ctc_greedy_decode()` 中跳过 `<blank>/<unk>/<pad>/<sos>/<eos>/<sil>` 等特殊 token；`_tokens_to_text()` 增加 regex 清理残留 `<xxx>` 标签。修复后最后一段从 64 字符（含大量 `<sil>`）降到 4 字符纯文本。
